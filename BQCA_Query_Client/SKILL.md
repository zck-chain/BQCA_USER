# BQCA 电商数据查询客户端

## 使用场景
- 用户以业务角色身份询问电商数据（订单、用户、商品等）
- 需要根据角色权限返回不同范围的数据
- 支持：一线客服（仅已发货订单）、运营经理（全量数据）

## 核心流程
1. 根据用户身份/角色判断使用哪个 API Key
2. 调用 BQCA 服务接口，传入自然语言问题
3. 解析响应数据，格式化展示给用户
4. 保存 conversation_id 用于多轮对话

## 输入
- 用户的问题（自然语言，如"查看订单各状态的分布情况"）
- 用户角色（从对话上下文推断或明确指定）

## API 调用方式

**端点**: `POST https://bqca-bot-839062387451.asia-east1.run.app/api/query`

**请求头**:
- `X-API-Key: <根据角色选择>`
- `Content-Type: application/json`

**请求体**:
```json
{"question": "自然语言问题"}
```

**多轮对话请求体**:
```json
{"question": "跟进问题", "conversation_id": "上轮返回的conversation_id"}
```

## 角色与权限

| 角色 | API Key | 数据范围 | 适用人群 |
|------|---------|---------|---------|
| 运营经理（全权限） | ${BQCA_MANAGER_API_KEY} | 所有订单状态（5种） | 运营经理、管理员、需要全量数据 |
| 一线客服（受限） | ${BQCA_SUPPORT_API_KEY} | 仅已发货订单（Shipped） | 一线客服、物流人员、仅看已发货 |

## 角色判断规则

出现以下关键词 → 选择对应 Key：

**运营经理（全权限）**: 运营经理、经理、管理员、全量、全部数据、所有状态

**一线客服（受限）**: 一线客服、客服、物流、仅发货、只看已发货

**未指定角色**: 默认使用运营经理（全权限）Key

## 响应字段说明

API 返回 JSON，包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| summary | string | BQCA 生成的完整分析报告，包含逻辑解释、SQL代码、业务洞察 |
| sql | string | 生成的 SQL 查询语句 |
| fields | array | 列名列表 |
| rows | array | 数据行，每行是 key-value 对象 |
| chart | boolean/object | 图表配置，false 表示无图表 |
| html_url | string | HTML 可视化页面链接 |
| conversation_id | string | 会话 ID，多轮对话时必须回传 |

## 结果处理规范

调用 API 后，必须对返回结果做以下处理，不要原样丢给用户：

### 1. 数据表格
将 `fields` 和 `rows` 格式化为 Markdown 表格展示。

### 2. 业务分析
从 `summary` 中提取关键信息呈现给用户：
- **逻辑解释**：查询的思路和方法
- **SQL 代码**：展示生成的 SQL
- **业务洞察**：数据背后的业务含义和建议

### 3. 补充计算
根据数据适当补充计算，提升分析价值：
- 分布类数据：计算各项占比（百分比）
- 趋势类数据：指出增减变化
- 对比类数据：标注差异显著项

### 4. 可视化链接
如果有 `html_url`，以可点击链接形式提供，如：[查看可视化报告](html_url)

### 5. 角色提示
如果用户使用受限角色，在结果中提示数据范围：
- 一线客服：提醒"当前仅展示已发货（Shipped）订单数据"
- 运营经理：无需额外提示

### 6. 多轮对话
保存 `conversation_id`，用户追问时回传以保持上下文连续。

## 输出形式

最终给用户的回复应包含：
1. **数据表格** — 格式化的查询结果
2. **业务洞察** — 从 summary 提取的关键分析和建议
3. **补充计算** — 占比、趋势等衍生指标（适用时）
4. **SQL 语句** — 生成的查询 SQL
5. **可视化链接** — HTML 报告页面

## 示例

### 一线客服查询
用户输入：我是一线客服，帮我查一下订单各状态的分布情况

→ 使用受限 Key → 调用 API → 结果只包含 Shipped 行
→ 展示时提示：当前仅展示已发货（Shipped）订单数据

### 运营经理查询
用户输入：我需要查看所有订单的状态分布

→ 使用全权限 Key → 调用 API → 结果包含全部 5 种订单状态
→ 展示表格 + 计算占比 + 业务洞察 + 可视化链接

### 多轮对话
用户第一轮：查看订单状态分布
用户第二轮：那上个月的退货情况呢？
→ 第二轮请求带上第一轮返回的 conversation_id
